Purpose

The purpose of this site is to allow users to explore the phenomenon of cellular automaton within the bounds of the Game of Life. The user can experiment with their own creations, view over 1500 previously created patterns, and apply different themes to create beautiful life forms.

Site Map

Here are some details about each of the tabs on the site.

  • Editor The Game of Life editor.
  • Patterns A massive list of Game of Life patterns. Over 1500! Don't forget to mouse over (or press for mobile) to preview the pattern.
  • Themes Themes gives you the ability to change how the Game of Life board looks. Be sure to turn on generations for some neat effects, but for larger patterns, it is best to turn this feature off.
  • About General details about the site, the Game of Life, and some interesting links.

Patterns

All patterns in the patterns section come from Life Wiki. The users of this site have spent a huge amount of effort creating and collecting these patterns. Without those patterns, this site would not exist. The list of patterns on this site was last updated from Life Wiki around June, 2017.

Development

I originally started on this site because I wanted to learn how to use the javascript framework Angular. It was never meant to be a fully operational website. Unfortunately, I broke my ankle playing soccer which gave me a lot more free time. Lacking inspiration I kept pushing this site further and further until it looks like the site you see today.

The Game of Life algorithm I used is called Tony Finch's List Life algorithm. I also used this site as inspiration for my choice of algorithm and used the site to help me through learning the best way to utilize the html canvas element.

For future development I would like the site to use the Hashlife algorithm since the list life algorithm doesn't handle large cellular automaton. I would also like to fix some of the loading times on the template page. I don't plan on adding a backend to this project. If you are interested in contributing to the site, check it out on GitHub.
